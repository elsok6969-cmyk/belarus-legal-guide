

# Страница просмотра документа `/doc/:slug`

## Обзор
Создать полнофункциональную страницу просмотра НПА с оглавлением, freemium-гейтом, связанными документами и счётчиком просмотров. Требуется миграция БД для добавления недостающих колонок.

## Шаг 1. Миграция базы данных

Добавить в таблицу `documents` недостающие колонки:
- `slug text UNIQUE` — для URL-адресов
- `organ text` — орган принятия
- `reg_number text` — регистрационный номер
- `reg_date date` — дата регистрации  
- `is_free boolean DEFAULT false` — бесплатный доступ
- `view_count integer DEFAULT 0` — счётчик просмотров

Сгенерировать slug из title для существующих записей (транслитерация кириллицы).

Создать RLS-политику, разрешающую публичный `UPDATE` только для `view_count` через database function (security definer) `increment_view_count(doc_id uuid)`.

## Шаг 2. Новая страница `src/pages/PublicDocumentView.tsx` (полная переработка)

**Шапка документа:**
- Хлебные крошки: Главная → Документы → [тип] → [название]
- Заголовок `text-2xl font-bold`
- Мета-строка: badge типа (цветовой: Кодекс=teal, Закон=blue, Указ=orange, Декрет=red, Постановление=purple), дата принятия, рег. номер, орган, badge статуса

**Кнопки действий:**
- Скачать PDF (tooltip «Доступно на тарифе Профи» для бесплатных)
- Поделиться (копировать URL → toast)
- В избранное (таблица `bookmarks`, уже есть)
- Следить за изменениями (таблица `subscriptions`, уже есть)

**Двухколоночный layout (md:flex):**

*Левая панель — Оглавление (sticky, w-64):*
- Парсинг body_text по regex: `Глава \d+`, `Статья \d+`, `Раздел [IVXLC]+`
- Якоря с `id="article-N"`, подсветка текущей секции через Intersection Observer
- На мобильном — FAB-кнопка «Содержание» → drawer

*Правая область — Текст документа:*
- Freemium-гейт: если `is_free=true` ИЛИ авторизован → полный текст. Иначе — первые 3 статьи + blur-sm overlay с CTA
- Форматирование: заголовки глав (`text-lg font-semibold mt-8`), номера статей (`text-base font-medium text-teal-700`), абзацы (`text-sm leading-relaxed`)

**Счётчик просмотров:** вызов `increment_view_count` при монтировании (один раз).

**Блок «История изменений»** (collapsible): данные из `document_versions` (уже есть таблица).

**Правая боковая панель (w-72, hidden lg:block):**
- Связанные документы: `documents WHERE doc_type = $1 AND id != $2 ORDER BY view_count DESC LIMIT 5`
- Заглушка «Экспертные комментарии» с полем email

**Мобильная адаптация:**
- Оглавление → FAB + full-screen drawer
- Кнопки действий → горизонтальный скролл
- Боковые панели → под текстом документа

## Шаг 3. Обновить маршрут в `App.tsx`

Маршрут `/doc/:slug` уже существует, но сейчас указывает на старый `PublicDocumentView`. Новая версия страницы будет загружать документ по slug: `.eq('slug', slug)`.

## Затрагиваемые файлы
- **Миграция SQL** — добавление колонок + функция `increment_view_count`
- `src/pages/PublicDocumentView.tsx` — полная переработка
- `src/App.tsx` — без изменений (маршрут уже есть)

## Технические детали
- Используем существующие таблицы `bookmarks` и `subscriptions` (вместо `user_favorites` / `user_subscriptions`)
- `document_versions` уже существует — используем для истории изменений
- Intersection Observer для подсветки текущей секции в оглавлении
- Sheet из shadcn/ui для мобильного drawer оглавления
- Clipboard API для «Поделиться»
